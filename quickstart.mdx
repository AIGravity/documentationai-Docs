## Get the site running

Create a basic docs structure, add your first pages, and confirm the site builds with the new navigation.

<Callout kind="info">

Set up your local environment before you start:

- Node.js and `npm`
- Git
- A docs content folder in your project, such as `docs` or `content`

</Callout>

## Create your first pages

<Steps>
  <Step title="Create `introduction.mdx`" icon="file-plus">

    Add a new page for the home or introduction section in your docs content folder.

    ```bash
    touch docs/introduction.mdx
    ```

    If your project uses a different content directory, replace `docs` with the folder your site reads from.

  </Step>

  <Step title="Add frontmatter" icon="edit">

    Open `introduction.mdx` and add the page metadata the site uses to build navigation and page titles.

    ```mdx
    ---
    title: Introduction
    description: Start here for an overview of the docs site and its main sections.
    ---
    ```

    Save the file and confirm the page now has a title and description at the top.

  </Step>

  <Step title="Create a second page" icon="plus">

    Add another page such as `features.mdx` so the navigation has more than one destination.

    ```bash
    touch docs/features.mdx
    ```

    Keep the second page in the same folder so it appears alongside the first page in your docs structure.

  </Step>

  <Step title="Add page content" icon="file-text">

    Add a short heading and a few sentences to each file so the pages render with visible content.

    ```mdx
    ## Introduction

    Welcome to the docs site.
    ```

    ```mdx
    ## Features

    Describe the main feature set here.
    ```

  </Step>
</Steps>

## Run and build the site

Use your project commands to start the local site and verify that the new pages compile.

<Steps>
  <Step title="Start the local preview" icon="play-circle">

    Run the development command your project uses.

    <CodeGroup tabs="npm">
    ```bash
    npm run dev
    ```
    </CodeGroup>

    If your project uses a different script name or package manager, substitute the equivalent command from your repo.

  </Step>

  <Step title="Build the site" icon="package">

    Run the production build after the preview starts successfully.

    <CodeGroup tabs="npm">
    ```bash
    npm run build
    ```
    </CodeGroup>

    A successful build confirms that the new pages and frontmatter parse correctly.

  </Step>

  <Step title="Confirm navigation" icon="check-circle">

    Open the site navigation and make sure both pages appear where you expect them.

    If the pages do not show up, check the content folder path, the file names, and any navigation configuration in your project.

  </Step>
</Steps>

## Verify the result

You should now be able to open both pages in the editor and in the running site.

<Callout kind="success">

You can open Introduction and Quickstart pages in the editor/site.

</Callout>
