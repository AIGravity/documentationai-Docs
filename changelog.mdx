## Overview

This changelog tracks user-facing changes that affect how people experience the Gravity web app and documentation surface. The newest release appears first, with updates listed before bug fixes.

<Callout kind="info">

This release did not add major new product features. It focuses on web app infrastructure, preserved theme behavior, and more useful routed error states.

</Callout>

## Release history

<Update label="Current" description="Web app infrastructure refresh" tags={["improvement", "bugfix"]}>

### Updates

- Refreshed the Remix web app shell and styling foundation to keep the app experience consistent across routes.
- Preserved the existing light and dark background behavior, including the white light-mode background and dark gray background for users who prefer dark mode.
- Kept this release focused on app shell readiness rather than major new product features.

### Bug fixes

- Improved routed error handling so missing pages and route errors show clearer, more specific messages instead of a generic failure state.
- Kept development-only error details available for debugging without exposing stack traces in normal routed error responses.

For implementation context, see [Frontend Application Shell](/frontend-application-shell).

</Update>
