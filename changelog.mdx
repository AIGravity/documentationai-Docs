## Latest changes

The latest `0.1.0` service baseline focuses on clearer release tracking and more practical setup guidance for teams running the Gravity monorepo.

<Update label="v0.1.0" description="Service baseline release" tags={["feature", "improvement"]}>

### New features

- **Service-level release tracking is now documented.** Gravity now documents independent `0.1.0` versions for the web app, API service, and AI service so teams can track compatibility and releases by service. See [Service Versioning](/service-versioning).

### Updates

- **Quickstart guidance now reflects the current monorepo workflow.** The setup path is clearer about local services, development commands, and the expected first successful run. See [Quickstart](/quickstart).
- **Architecture guidance now gives a clearer map of the platform.** The overview connects the frontend, API service, AI service, and shared packages so readers can understand how the system fits together. See [Architecture Overview](/architecture-overview).
- **Frontend shell documentation now matches the current app baseline.** The frontend docs describe the application shell, routing layer, styling approach, and developer experience expected in the web app. See [Frontend Application Shell](/frontend-application-shell).
- **Health check guidance is easier to use during setup and operations.** The docs call out the local service endpoints teams use to confirm that the platform is running correctly. See [Health Checks](/health-checks).

### Bug fixes

- No user-facing bug fixes shipped in this update.

</Update>
